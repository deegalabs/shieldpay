# Equity Analysis

> Phase: audit | Brand: ShieldPay | Generated: 2026-06-27

---

Recognition value per element, and the harder call: which assets carry genuine
equity worth protecting versus which persist only by inertia and could change
with no loss.

## Element-by-element equity

| Element | Recognition | Genuine equity or inertia? |
| --- | --- | --- |
| Indigo to emerald gradient | High | **Genuine.** Distinctive, story-bearing, ownable. |
| Shield-check mark | Medium | **Genuine but generic.** Equity in "shield = protected"; the specific glyph is near-stock lucide. |
| Calm dark slate palette | High | **Genuine.** Correct, on-trend, category-appropriate. |
| Emerald "verified" badge | Medium | **Genuine.** The status chip is a real recognizable moment. |
| Inter typeface | Low | **Inertia.** Correct default, zero distinctiveness. |
| Mono-for-figures intent | Low (today) | **Latent equity.** Strategically owned, not yet visible. |
| "Cryptography is invisible" rule | High (internal) | **Genuine strategic equity.** Drives the whole brand. |
| Flat surface/elevation system | n/a | **Pure inertia / liability.** Nothing to protect. |
| `accent` token (= brand) | n/a | **Inertia.** Dead duplicate. |
| Ambient corner glows | Low | **Weak.** Pleasant but invisible; not yet an asset. |

## The genuine equity to protect

**1. The indigo-to-emerald gradient (high).** This is the brand's crown jewel. It
is uncommon in fintech (most competitors are mono-accent), it is semantically
loaded ("protected, then proven"), and it ties identity, mark, and hero into one
idea. It maps cleanly to the personas: indigo reassures the privacy-anxious CFO,
emerald confirms settlement for the worker and auditor. **Preserve and elevate**
(use it as dimensional light, not flat fill). Replacing it would discard the only
thing that makes ShieldPay visually un-generic.

**2. The calm dark slate palette (high).** The slate base is correct, current, and
exactly what makes a skeptical CFO trust a crypto-adjacent tool. It is equity
because it actively differentiates from neon crypto. **Preserve the hue family;
evolve the structure** (add elevation steps, a mid text tier) without changing
what the palette *feels* like.

**3. The "verified" emerald badge (medium).** Across three portals the green
"Verified on-chain" chip is the recurring proof-of-trust moment, the visual
payoff of the whole product. It is a small but real recognition asset for all
three personas. **Preserve the meaning; evolve the finish.**

**4. "Cryptography is invisible" (high, internal).** Not a visual mark but the
strongest equity the brand has: a clear, enforced principle that separates
ShieldPay from every hacker-aesthetic crypto tool. **Preserve absolutely.**

## Equity that is real but generic (evolve, do not lean on)

**The shield-check mark (medium).** Users do read "shield = protected," so the
*concept* has equity. But the current glyph is close to stock lucide `ShieldCheck`
geometry; IDENTITY.md itself flags this and proposes a check-as-negative-space
variant. The equity lives in the shield idea and the gradient it carries, not in
the exact path. **Evolve the glyph** to something distinct (the negative-space
privacy-slit concept) while keeping the silhouette and gradient so recognition
survives.

## Inertia masquerading as identity (safe to replace)

- **The flat elevation model** has no recognition value and is the chief craft
  liability. Replacing it with a real surface-light system loses nothing.
- **The `accent` token** is a dead duplicate of `brand`; collapse or repurpose it.
- **Inter** is a sensible default but carries no equity; it can stay for cost
  reasons or be paired with a more characterful display/mono face with no loss of
  recognition.
- **The legacy metadata tagline** is stale positioning inertia; replace it with
  the current confidential-payroll line.

## Net

The brand's durable equity is small but high-quality: the gradient, the slate
calm, the verified badge, and the invisible-cryptography principle. Everything
else is either neutral default (Inter, the generic shield) or unfinished
scaffolding (flat surfaces, dead tokens). This is the ideal shape for an
*evolution*: protect four real assets, evolve the mark and palette structure,
replace only the inert plumbing.
